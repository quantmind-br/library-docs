## Architecture Overview

**UI Dojo** is a comprehensive Mastra showcase project that demonstrates integrations between [Mastra](https://mastra.ai/) (an AI orchestration framework) and three major AI UI frameworks: **AI SDK**, **Assistant UI**, and **CopilotKit**. It's a reference implementation that allows developers to compare different approaches to building AI-powered interfaces and choose the best fit for their needs.

### Purpose & Use Case

The project serves as a **learning and reference platform** for developers building AI applications. Rather than providing a single solution, it showcases multiple patterns and architectural approaches side-by-side, making it ideal for:

- Understanding how different UI frameworks integrate with Mastra
- Learning AI workflow patterns (streaming, suspend/resume, multi-agent systems)
- Comparing implementation strategies for similar features
- Exploring advanced patterns like generative UIs and agent networks

## Project Structure

The codebase is organized into three primary layers:

### 1. **Frontend Layer** (`src/pages/` & `src/components/`)

#### Page Categories:

- [**Chat Examples**](https://github.com/mastra-ai/ui-dojo/blob/main/src/pages/) - Three parallel implementations showing identical chat functionality with different frameworks:
  
  - [**AI SDK**](https://github.com/mastra-ai/ui-dojo/blob/main/src/pages/ai-sdk/index.tsx) - Vercel's AI SDK with `@mastra/ai-sdk`
  - **Assistant UI** - Using Thread components with `useExternalStoreRuntime()` to connect to Mastra's memory
  - **CopilotKit** - Built with CopilotKit's Chat component
- [**AI SDK Advanced Features**](https://github.com/mastra-ai/ui-dojo/tree/main/src/pages/ai-sdk):
  
  - [Generative User Interfaces](https://github.com/mastra-ai/ui-dojo/blob/main/src/pages/ai-sdk/generative-user-interfaces.tsx) - Custom UI components for tool responses (e.g., dynamic weather cards with proper icons and formatting)
  - [Workflows](https://github.com/mastra-ai/ui-dojo/blob/main/src/pages/ai-sdk/workflow.tsx) - Multi-step workflows like activity planning
  - [Agent Networks](https://github.com/mastra-ai/ui-dojo/blob/main/src/pages/ai-sdk/network.tsx) - Multiple agents coordinating through routing
  - **Suspend/Resume** - Human-in-the-loop approval patterns
  - **Custom Events** - Advanced event handling across agents and workflows
- [**Client Tools**](https://github.com/mastra-ai/ui-dojo/tree/main/src/pages/client-tools) - Demonstrates client-side tool execution with all three frameworks

#### UI Components:

- [**`ai-elements/`**](https://github.com/mastra-ai/ui-dojo/tree/main/src/components/ai-elements) - AI-specific components:
  
  - Conversation, Message, Response
  - PromptInput with model selection, attachments, tools
  - Sources, Reasoning, Suggestions
  - Tool and ToolOutput displays
- [**`ui/`**](https://github.com/mastra-ai/ui-dojo/tree/main/src/components/ui) - Radix UI-based components with Tailwind styling

### 3. **API Routes**

[Configured in `src/mastra/index.ts`](https://github.com/mastra-ai/ui-dojo/blob/main/src/mastra/index.ts#L58-L80):

```
GET /chat/:agentId           → Chat with specific agent
GET /workflow/:workflowId     → Execute workflow
GET /network                  → Network routing (ghibli + weather agents)
GET /network-custom-events    → Report agent network with custom events
GET /copilotkit              → CopilotKit runtime
```

## Key Features & Patterns

### 1. **Framework Comparison**

Side-by-side implementations showing how [AI SDK](https://github.com/mastra-ai/ui-dojo/blob/main/src/pages/ai-sdk/index.tsx), Assistant UI, and CopilotKit each approach the same problem with different capabilities and trade-offs.

### 2. **Generative UIs**

[Dynamic UI generation](https://github.com/mastra-ai/ui-dojo/blob/main/src/pages/ai-sdk/generative-user-interfaces.tsx) where tool responses render custom React components. Example: Weather API calls return structured data rendered as styled weather cards with dynamic icons.

### 3. **Multi-Step Workflows**

Workflows coordinate multiple steps with typed inputs/outputs using Zod schemas. [Activities workflow](https://github.com/mastra-ai/ui-dojo/blob/main/src/mastra/workflows/activities-workflow.ts) chains weather fetching → activity planning with streaming support.

### 4. **Agent Networks**

[Multiple agents coordinate](https://github.com/mastra-ai/ui-dojo/blob/main/src/mastra/agents/network-routing-agent.ts) through routing agents, allowing complex multi-agent orchestration.

### 5. **Human-in-the-Loop**

[Approval workflow](https://github.com/mastra-ai/ui-dojo/blob/main/src/mastra/workflows/approval-workflow.ts) demonstrates suspend/resume patterns where workflows pause waiting for human approval, then resume.

### 6. **Client Tools**

Tools can execute on the client side (like `changeBgColor`) rather than server-side, with integration across all three UI frameworks.

### 7. **Streaming & Partial Responses**

Built-in support for streaming agent/workflow responses and handling partial tool execution states.

## Tech Stack

- Mastra 0.18.1 (AI orchestration)

- LibSQL (in-memory storage)

**Development:**

- TypeScript 5.9
- ESLint + Prettier
- Concurrent dev servers (Mastra + Vite)
