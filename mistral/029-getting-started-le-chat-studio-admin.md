Mistral AI has four interfaces, each designed for a different role.

| Interface | Purpose | Audience |
|---|---|---|
| **Le Chat** | AI workspace for chat, search, document analysis, and agents | Business users |
| **Studio** | API keys, model testing, fine-tuning, evaluation | Developers |
| **Mistral Vibe** | Terminal-native coding agent powered by Devstral | Developers |
| **Admin** | Organization setup, billing, SSO, team management | IT/Security |

## Le Chat

AI workspace for chat, search, document analysis, Canvas editing, and custom agents. No code required.

[Open Le Chat](https://chat.mistral.ai)

## Studio

Build and deploy Agents, process documents and audio, test prompts in the Playground, manage API keys, and monitor usage. Everything needed to build, ship, and operate AI applications on the Mistral API.

[Open Studio](https://console.mistral.ai)

## Mistral Vibe

Open-source, terminal-native coding agent powered by Devstral. Scan file structure and Git state, maintain multi-file context, run shell commands directly from terminal. Integrates with JetBrains, Zed, and any IDE supporting the Agent Communication Protocol. Can run fully offline with local models.

[Try Mistral Vibe](https://github.com/mistralai/mistral-vibe)

## Admin

Control plane for IT and security teams. Manage organization account, team structure, billing, SSO, and security policies.

[Open Admin](https://admin.mistral.ai)

Admins have full control over organization settings, users, security, and billing. The Billing role is an optional, limited role for finance teams who only need access to subscriptions, invoices, and payment methods.
