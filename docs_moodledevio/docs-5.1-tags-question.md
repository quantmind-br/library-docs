## [Defaults for new questions](https://moodledev.io/docs/5.1/apis/plugintypes/qtype/newquestiondefaults)

A way for question types to remember a user's preferred settings for creating questions of a given type.

## [Question bank filters](https://moodledev.io/docs/5.1/apis/plugintypes/qbank/filters)

Question bank plugins allow you to define new filters for the question bank view and random question sets.

## [Question bank plugins](https://moodledev.io/docs/5.1/apis/plugintypes/qbank)

Question bank plugins allow you to extend the functionality of the Moodle Question bank.

## [Question type plugin restore code](https://moodledev.io/docs/5.1/apis/plugintypes/qtype/restore)

Question type plugins must implement special restore code to avoid duplicated questions

## [Question type plugins](https://moodledev.io/docs/5.1/apis/plugintypes/qtype)

Question type plugins implement the different types of question that the core Question subsystem can handle.

## [Questions API](https://moodledev.io/docs/5.1/apis/subsystems/question)

The question subsystem in Moodle manages the creation, editing and management of interactive questions, and then enables those questions to be presented to users so they can be attempted. It is useful to consider these as separate subcomponents.
