## 概要

Firecrawl は **クレジットベースの課金システム**を採用しています。行う API 呼び出しごとにクレジットが消費され、消費されるクレジット数は利用するエンドポイントやオプションによって異なります。プランごとに毎月のクレジット枠が決まっており、必要に応じて自動チャージパックで追加クレジットを購入できます。 現在のプラン料金については、[Firecrawl の料金ページ](https://www.firecrawl.dev/pricing)を参照してください。

## クレジット

クレジットは、Firecrawl における利用を測る単位です。各プランには毎月のクレジット割り当てが含まれており、請求サイクルの開始時にリセットされます。API の各エンドポイントごとに、消費するクレジット数は異なります。

### エンドポイントごとのクレジットコスト

### スクレイピングのオプションによる追加クレジットコスト

一部のスクレイピングのオプションは、ページごとの基本コストに加えてクレジットが加算されます:

これらの修飾子は重複して適用されます。たとえば、JSON format と Enhanced Mode の両方を使用して 1 ページをスクレイピングすると、そのページのコストは **1 + 4 + 4 = 9 クレジット** になります。これらと同じ修飾子は、各ページに対して内部的にスクレイピングを使用するため、クロールおよび Search エンドポイントにも適用されます。

### クレジットが消費されるタイミング

**batch scrape** や **crawl** ジョブの場合、クレジットはジョブの送信時ではなく、各ページの処理が完了するたびに非同期で課金されます。つまり、ジョブを送信してから、アカウント上に最終的なクレジット消費が反映されるまでにタイムラグが生じる場合があります。バッチに多くの URL が含まれている場合や、高トラフィック時にページがキューに入れられる場合、送信後も数分から数時間にわたってクレジットが順次反映され続けることがあります。バッチのステータスをポーリングまたは確認しても、クレジットは消費されません。

### 利用状況の確認

クレジットの利用状況は、次の2つの方法で確認できます。

- **ダッシュボード**: 現在および過去の利用状況を [firecrawl.dev/app](https://www.firecrawl.dev/app) で確認できます
- **API**: [Credit Usage](https://docs.firecrawl.dev/ja/api-reference/endpoint/credit-usage) および [Credit Usage Historical](https://docs.firecrawl.dev/ja/api-reference/endpoint/credit-usage-historical) エンドポイントを使用して、プログラムから利用状況を確認できます

## プラン

Firecrawl はサブスクリプション制の月額プランを提供しています。純粋な従量課金プランはありませんが、後述の自動チャージ機能により、ベースプランに対して柔軟にスケールさせることができます。

すべての有料プランは **月払い** または **年払い** に対応しています。年払いは月払いと比べて割引があります。各プランの最新の料金については、[料金ページ](https://www.firecrawl.dev/pricing) を参照してください。

### 同時ブラウザ数

同時ブラウザ数は、Firecrawl が同時に処理できる Web ページ数を表します。この上限はご利用プランによって決まります。上限を超えた場合、追加のジョブは空きが出るまでキューで待機します。同時実行数と API のレート制限の詳細については、[Rate Limits](https://docs.firecrawl.dev/ja/rate-limits) を参照してください。

## 自動チャージ

ときどきプランに含まれるクレジット以上に利用する必要がある場合は、ダッシュボードから **自動チャージ** を有効にできます。残りクレジットが設定したしきい値を下回ると、Firecrawl が自動的に追加のクレジットパックを購入し、残高に加算します。

- 自動チャージ用のパックは、すべての有料プランで利用できます
- パックのサイズと価格はプランごとに異なります ([料金ページ](https://www.firecrawl.dev/pricing) で確認できます)
- 自動チャージのしきい値を設定でき、自動チャージはいつでもオン／オフを切り替えられます
- 自動チャージは **月あたり4パックまで** に制限されています
- 自動チャージ用パックのクレジットは **毎月リセットされません** — 月次プランのクレジットが各期間ごとにリセットされるのとは異なり、請求サイクルをまたいで保持され、購入から **1年後** に失効します。

自動チャージは、利用量が一時的にスパイクするケースへの対応に最適です。継続的にプランの割り当てを超えてしまう場合は、より上位のプランにアップグレードした方が、クレジットあたりのコスト面で有利になります。

## クーポン

Firecrawl は、2種類のクーポンに対応しています。

- **サブスクリプションクーポン**は、プランのサブスクリプション料金に割引を適用します (例: 月額または年額料金の一定割合の割引) 。これらは、有料プランに初めて登録する際、またはプラン変更時の Stripe のチェックアウトフロー中に**のみ**適用できます。チェックアウト完了後にサブスクリプションクーポンを適用することはできません。
- **クレジットクーポン**は、アカウントにボーナスクレジットを追加します。これらは、ダッシュボードの**課金**セクションにある [firecrawl.dev/app/billing](https://www.firecrawl.dev/app/billing) から引き換えできます。コードを適用するには、課金ページのクーポン入力欄を探してください。クレジットクーポンによるボーナスクレジットは、プランの月次割り当てとは別であり、プランをアップグレードまたはダウングレードしても保持されます。

クーポンコードをお持ちで、どちらの種類かわからない場合は、まずダッシュボードの課金セクションで適用してみてください。サブスクリプションクーポンの場合は、代わりに Stripe のチェックアウトページで使用する必要があります。

## 請求サイクル

- **月額プラン**: クレジットは毎月の更新日にリセットされます
- **年額プラン**: 請求は年に一度ですが、クレジットはシステム上の毎月の更新日にリセットされます
- **未使用のプランのクレジットは繰り越されません** — 各請求期間の開始時に、毎月の割り当てはプランで定められたクレジット数にリセットされます。自動チャージパックのクレジットは請求サイクルに紐づいていません — 購入日から**1年**後に失効するまで保持されます。

## アップグレードとダウングレード

- **アップグレード** は即時に反映されます。現在の請求期間の残り日数に応じた差額が請求され、クレジットの割り当てと上限はすぐに更新されます。
- **ダウングレード** は次回の更新日に反映されるようにスケジュールされます。それまでは現在のプランのクレジットと上限を維持します。

## クレジットを使い切った場合

クレジット残高を使い切り、かつ自動チャージを有効にしていない場合、クレジットを消費する API リクエストは **HTTP 402 (Payment Required)** エラーを返します。自動チャージを有効にしている場合は、自動的にリチャージパックが購入されるあいだは利用を継続できますが、月間パック上限に達した場合やリチャージに失敗した場合は、次の請求サイクルまたは手動でチャージするまで残高がマイナスになることがあります。強制停止後に利用を再開するには、次のいずれかを行ってください。

1. 自動チャージを有効にしてクレジットを自動購入する
2. 上位プランにアップグレードする
3. 次の請求サイクルでクレジットがリセットされるのを待つ

## 無料プラン

無料プランでは、**一度きりで 500 クレジットが付与**され、クレジットカードは不要です。これらのクレジットは更新されないため、使い切った後も Firecrawl を継続利用するには有料プランへのアップグレードが必要です。無料プランは、有料プランと比べてレート制限および同時実行数が低くなっています ([レート制限](https://docs.firecrawl.dev/ja/rate-limits) を参照) 。
