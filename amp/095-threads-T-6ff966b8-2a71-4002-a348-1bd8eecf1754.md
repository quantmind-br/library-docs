I want to add a new agent. But it's different from the rest. It runs in parallel to the main thread. on every inference:completed (either tool\_use or or end\_turn) it runs one forced tool call with gemini with the entire thread. The forced tool call is to a new tool we are adding called course\_correct, course correct has a boolean field (if true we want to add a course correct), and an optional message to be sent to the agent to stear it. The message should be like a user message, but it should have some metadata so we can render it differently. You should use gemini 3 as the model. The system prompt should ask it to check whether the model has forgotten anything or whether the model is going too far off the plan (diverging) or too narrow (not doing enough) and then correct its course. Or whether anything was forgotten based on the AGENTS.md file (run lint, ...) We need to: 1. add the tool 2. add the logic to trigger the forced tool call (look at the handoff logic for this for how to force a tool call with a prompt) 3. Create a good system prompt for it (we want the custom system prompt, the messages from the original thread as markdown, which should hightlight the user messages and the AGENTS.md file read in). 4. Add the logic to send the user:message in the thread worker / thread delta based on the result. This should run in the background once an tool\_use stop reason is here, so it completed with the rool results and it should be sent after the tool results have been added to the thread but before we send a new request. On stop reason end of turn, we run it and send it only if the user has not sent a message yet.
