The diagram shows the complete flow:

1. **Entry** → User clicks invite link, hits `/api/signup/{invite_id}`
2. **Validation** → Email match, expiration (72h), existing membership checks
3. **User Phase** → Create new user or get authenticated user
4. **invite.use() Core Logic** → Creates `OrganizationMembership`, sets current org/team, assigns default role, updates billing
5. **AccessControl** → Creates per-project permission records for private projects
6. **Product Sync** → Backfills user preferences from colleagues
7. **Notifications** → Sends member join email, deletes pending invites
8. **Analytics** → Reports signup/join events, aliases invite ID
