## PostHog Invitation System - Data Model and Architecture

Based on my analysis of the PostHog codebase, here's a comprehensive overview of how the invitation system works:

### **Core Models**

#### [OrganizationInvite](https://github.com/PostHog/posthog/blob/main/posthog/models/organization_invite.py)

The central model for managing invitations:

```
class OrganizationInvite(ModelActivityMixin, UUIDTModel):
    organization = ForeignKey(Organization, CASCADE)  # Organization being invited to
    target_email = EmailField()                        # Email of invitee
    first_name = CharField(max_length=30)              # Invitee's first name
    created_by = ForeignKey(User, SET_NULL)            # Who sent the invite
    emailing_attempt_made = BooleanField()             # Email tracking
    created_at = DateTimeField(auto_now_add=True)      # When created
    updated_at = DateTimeField(auto_now=True)          # Last update
    message = TextField()                               # Optional custom message
    level = PositiveSmallIntegerField()                # Role level (MEMBER=1, ADMIN=8, OWNER=15)
    private_project_access = JSONField()               # List of teams with [{"id": <team_id>, "level": "member|admin"}]
```

**Key Properties:**

- **Validity**: Invites expire after `INVITE_DAYS_VALIDITY` (3 days) - checked via `is_expired()`
- **Activity Tracking**: Inherits from `ModelActivityMixin` to track all invite changes in activity logs
- **Access Control**: Stores both organization-level role and fine-grained project-level access

### **Invitation Flow: Creation → Sending → Acceptance**

#### 1. **Creation** - [OrganizationInviteViewSet.create()](https://github.com/PostHog/posthog/blob/main/posthog/api/organization_invite.py#L338-L348)

```
# API endpoint: POST /api/organizations/{org_id}/invites/
POST_DATA = {
    "target_email": "user@example.com",
    "first_name": "John",  # optional
    "level": 1,  # 1=MEMBER, 8=ADMIN, 15=OWNER
    "message": "Custom message",  # optional
    "private_project_access": [  # optional
        {"id": <team_id>, "level": "member"}  # or "admin"
    ],
    "send_email": True,  # default
    "combine_pending_invites": False  # optionally merge with existing
}
```

**Creation Process:**

1. Validates user can invite (based on `members_can_invite` permission)
2. Checks inviter's level vs invitee's level (can't invite higher)
3. Optionally combines with existing pending invites for same email
4. Deletes previous invites for same email
5. Creates new `OrganizationInvite` record
6. Logs activity via `model_activity_signal`

#### 2. **Sending Email** - [send\_invite()](https://github.com/PostHog/posthog/blob/main/posthog/tasks/email.py#L135-L158)

```
@shared_task
def send_invite(invite_id: str) -> None:
    invite = OrganizationInvite.objects.select_related(
        "created_by", "organization"
    ).get(id=invite_id)
    
    message = EmailMessage(
        use_http=True,
        campaign_key=f"invite_email_{invite_id}",
        subject=f"{invite.created_by.first_name} invited you to "
                f"{invite.organization.name} on PostHog",
        template_name="invite",
        template_context={
            "invite": invite,
            "expiry_date": timezone.now() + timedelta(days=3),
            "url": f"{SITE_URL}/signup/{invite_id}",
        }
    )
    message.add_recipient(
        email=invite.target_email, 
        distinct_id=f"invite_{invite_id}"  # Aliased after signup
    )
    message.send()
```

#### 3. **Acceptance** - [OrganizationInvite.use()](https://github.com/PostHog/posthog/blob/main/posthog/models/organization_invite.py#L125-L163)

```
def use(self, user: User, *, prevalidated: bool = False) -> None:
    # 1. Validate invite (not expired, right email, etc.)
    if not prevalidated:
        self.validate(user=user)
    
    # 2. Join user to organization with specified level
    user.join(organization=self.organization, level=self.level)
    
    # 3. Grant project-specific access if specified
    for item in self.private_project_access or []:
        team = self.organization.teams.get(id=item["id"])
        parent_membership = OrganizationMembership.objects.get(
            organization=self.organization,
            user=user,
        )
        AccessControl.objects.create(
            team=team,
            resource="project",
            resource_id=str(team.id),
            organization_member=parent_membership,
            access_level=item["level"],  # "member" or "admin"
        )
    
    # 4. Sync user's product list for accessible teams
    self._sync_user_product_list_for_accessible_teams(user)
    
    # 5. Send "member join" notification to other members
    if self.organization.is_member_join_email_enabled:
        send_member_join.apply_async(kwargs={
            "invitee_uuid": user.uuid,
            "organization_id": self.organization_id,
        })
    
    # 6. Delete all invites for this email (cleanup)
    OrganizationInvite.objects.filter(
        organization=self.organization,
        target_email__iexact=self.target_email
    ).delete()
```

### **Activity Logging**

Invitations are tracked via `ModelActivityMixin` which logs:

- **Invite creation**: `activity="created"`
- **Invite updates**: Field changes logged in detail
- **Invite deletion**: `activity="deleted"`
- **Context captured**: Organization ID, invitee email, inviter name

Activity logs appear in the organization's activity timeline and can be viewed via the API.
